# 콘텐츠 자산 수집 체크리스트

제작 전에 아래 항목을 모으면 페이지 작업이 빨라지고, 오픈 후 수정도 줄어듭니다.

## 브랜드·기본 정보

- [ ] 로고 원본(PNG/SVG, 배경 투명 권장) — 현재: `영전사_로고-removebg-preview.png`
- [ ] 대표 색상 코드(예: `#c62828`) — `index.html` `:root`에 반영됨
- [ ] 상호, 대표자명, 사업자등록번호, 본사 주소, 대표전화, 대표 이메일
- [ ] 카피라이트 연도 표기 방식

## 기업소개용

- [ ] 경영이념·목표 문구(1~3문단 + 핵심 키워드 3~5개)
- [ ] 비전·미션(중장기 방향 한 페이지 분량)
- [ ] 연혁 표(년도 / 내용) — 엑셀 후 HTML 표로 옮기기
- [ ] 조직도 이미지(PNG) 또는 부서 목록 텍스트
- [ ] 자격·인증서 스캔본(PDF 또는 JPG, 파일명 규칙 통일)

## 공사실적·갤러리

- [ ] 분야별(전기 / 소방 / 통신) 대표 프로젝트 목록: 공사명, 발주처(가능 시), 기간, 한 줄 설명
- [ ] 현장·완공 사진(가로 1200px 이상 권장, 용량 300KB 내외로 WebP·압축)
- [ ] 사진 사용 동의(촬영지·건물 외관 등)

## 커뮤니티

- [ ] 공지/소식 글 템플릿: 제목, 날짜, 요약 1줄, 본문
- [ ] 자료실 파일: 브로셔 PDF, 회사소개서, 인증서 모음 ZIP 등
- [ ] 온라인 문의 수신 담당자·내부 처리 절차(누가 몇 시간 내 회신할지)

## 인재채용

- [ ] 채용공고 문구(모집 분야, 자격요건, 지원 방법)
- [ ] 입사지원 시 필요 서류(이력서, 경력기술서, 자격증 사본 등)
- [ ] 지원 접수 이메일 또는 구글폼 URL(선택)

## 법·정책

- [ ] 개인정보처리방침 초안(수집 항목, 목적, 보관 기간, 문의 창구)
- [ ] 문의 폼 개인정보 수집·이용 동의 문구

## 현재 저장소에 넣는 위치(권장)

| 자산 종류 | 경로 |
|-----------|------|
| 이미지 | `assets/img/` 하위 폴더 구분 (`gallery/`, `cert/`, `org/` 등) |
| PDF | `assets/files/` (폴더 생성 후 업로드) |
| 본문 초안 | `docs/drafts/` (마크다운·워드 복사용) |

## 정적 HTML 한계(알아두면 좋음)

- 지금 구조는 **게시판이 “HTML을 직접 수정”**하는 방식에 가깝습니다.
- 비개발자가 글만 올리려면 나중에 **아임웹·워드프레스·노션+도메인** 등으로 이관하거나, 간단한 **헤드리스 CMS**를 붙이는 편이 운영에 유리합니다.
